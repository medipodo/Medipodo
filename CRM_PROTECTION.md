# CRM Protection Guide

The CRM (`/crm` page and `/api/crm/*` endpoints) currently has **no in-app authentication**. This is intentional: the routes are isolated so you can put any protection you like in front of them without touching the app.

This document shows the three most common ways to lock the CRM down at the edge. Pick one — they're all production-grade.

---

## What's already done in the app

| Piece                  | Detail                                                                                                                   |
|------------------------|--------------------------------------------------------------------------------------------------------------------------|
| Single URL prefix      | All CRM API routes live under `/api/crm`. The CRM page lives at `/crm`. Protecting **those two paths** covers everything. |
| Dedicated router       | `backend/crm_router.py` — completely separate from the public form. No CRM endpoint can accidentally land under `/api/`.  |
| Auth seam ready        | Every CRM endpoint declares `dependencies=[Depends(require_admin)]`. Replace the body of `require_admin` and you have in-app auth — zero other code changes. See `crm_router.py`. |
| OpenAPI tag            | All CRM endpoints carry the `crm` tag, so a future "protect everything tagged `crm`" middleware is one-liner away.        |
| No mixed payloads      | The public form endpoint (`/api/assessment-requests`) is in `server.py`; CRM endpoints are in `crm_router.py`. Different files, different routers, different protection profiles. |

## Option 1 — Nginx Basic Auth (simplest)

Two `htpasswd` lines and one `location` block. Good for an internal team of 2-3 podologists.

```bash
# create the credentials file once
sudo apt-get install -y apache2-utils
sudo htpasswd -c /etc/nginx/.medipodo_admins drrana
sudo htpasswd    /etc/nginx/.medipodo_admins drmehmet
```

```nginx
# /etc/nginx/sites-available/medipodo
server {
    listen 443 ssl http2;
    server_name medipodo.com;

    # ----- public site (unprotected) -----
    location / {
        proxy_pass http://app_upstream;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # ----- CRM page and API: locked down -----
    location ~ ^/(crm|api/crm) {
        auth_basic           "Medipodo CRM";
        auth_basic_user_file /etc/nginx/.medipodo_admins;

        proxy_pass http://app_upstream;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        # Allow the auth header to flow through if needed
        proxy_set_header Authorization $http_authorization;
    }
}
```

Reload Nginx and the CRM is protected — both the React page **and** the API. Anyone trying `/api/crm/assessment-requests` without credentials gets a 401 before the app ever sees the request.

## Option 2 — Cloudflare Zero Trust (best for small teams, free up to 50 users)

No password files, full audit log, SSO with Google / Microsoft / one-time email codes.

1. **Tunnel / DNS** — Make sure `medipodo.com` proxies through Cloudflare (orange cloud).
2. **Cloudflare Dashboard → Zero Trust → Access → Applications → Add application → Self-hosted.**
3. Configure:
   - **Application name**: `Medipodo CRM`
   - **Session duration**: 24 h
   - **Application domain**: `medipodo.com`
   - **Path** (add two):
     - `crm`
     - `api/crm/*`
4. **Policies → Add policy**:
   - **Action**: `Allow`
   - **Rules**: Emails → `drrana@medipodo.com`, `drmehmet@medipodo.com` (or whole `@medipodo.com` domain).
5. (Optional) Add a second policy with `Action: Block` and `Everyone` to be explicit.

Done. Visiting `/crm` redirects to Cloudflare's identity prompt; only allow-listed emails get through. The same protection covers every `/api/crm/*` call.

> Bonus: enable `Service Tokens` if you ever want a server-to-server CRM integration — Cloudflare hands you a header pair (`CF-Access-Client-Id` / `CF-Access-Client-Secret`) that bypasses the human auth.

## Option 3 — Caddy with Basic Auth

If you're on Caddy instead of Nginx:

```caddy
medipodo.com {
    encode zstd gzip

    @crm path /crm /crm/* /api/crm /api/crm/*
    handle @crm {
        basicauth {
            drrana   JDJhJDE0JDF...   # generated by: caddy hash-password
            drmehmet JDJhJDE0JDF...
        }
        reverse_proxy app_upstream:80
    }

    handle {
        reverse_proxy app_upstream:80
    }
}
```

## Option 4 — Tailscale / VPN (zero-config, internal only)

If your podologists work from the clinic + their home laptops, the simplest possible setup is putting the whole app on a private network:

1. Install Tailscale on the server.
2. Bind the proxy to the Tailscale IP only.
3. Add each podologist's device to the Tailnet.

Nothing else to configure. The CRM is only reachable from devices on your Tailnet.

## When you want in-app auth later (no proxy)

If you decide to ship auth inside the app (Supabase Auth, custom JWT, signed cookie, etc.), you only need to edit **one function** — `require_admin` in `backend/crm_router.py`:

```python
def require_admin(request: Request) -> None:
    token = request.headers.get("authorization", "").removeprefix("Bearer ").strip()
    if not token or not is_valid_admin_token(token):
        raise HTTPException(status_code=401, detail="Unauthorized")
```

Every CRM endpoint already declares `Depends(require_admin)` so they'll all start enforcing immediately. No route signatures change. No frontend changes needed beyond attaching the token header.

For the frontend, the existing `apiFetch` helper in `pages/CRM.jsx` is the single point where you'd add the `Authorization` header — one line.

## Quick verification

After applying any of the options above, this curl should be rejected (401 / 403) **before** reaching the app:

```bash
curl -i https://medipodo.com/api/crm/assessment-requests
```

And the public form must still work:

```bash
curl -i -X POST https://medipodo.com/api/assessment-requests \
  -F "phone=05551112233" -F "complaint=test" -F "foot=Sağ" -F "kvkk_accepted=true"
```

If both behave that way, you're done.
